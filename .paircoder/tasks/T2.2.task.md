# Objective

Implement the 10-step allocation engine (income selection, fixed obligations, deficit trigger, per-bucket distribution, floors, rounding residue → Savings, snapshot, bill_status creation) and the provisional week creation path. This is the most critical business logic file.

# Implementation Plan

Split into 5 files (each ≤ 3-4 exported functions to stay under limits):

- `src/lib/engine/week.ts`:
  - `getMostRecentFriday(date)`: returns most recent Friday ≤ date
  - `createProvisionalWeek(userId)`: creates week with baseline_weekly_income, status=projected
  - `reassignFridayTransactions(weekId, friday)`: moves same-day transactions from prior week
  - `promoteProvisionalWeek(weekId, depositData)`: upgrades projected → active

- `src/lib/engine/allocation.ts` (orchestrator):
  - `runAllocationEngine(weekId, income)`: calls sub-functions in order; wraps all DB writes in transaction
  - `computeDistributable(income, billTotal)`: income - sum(active bill amounts)
  - `distributeByPct(distributable, buckets)`: floors down to cent, returns allocations map

- `src/lib/engine/deficit.ts`:
  - `checkDeficitTrigger(income, distributable, foodMin)`: Condition A + B; returns `{ deficit, condition }`
  - Three plan computation functions (see T2.5)

- `src/lib/engine/daily-limit.ts`:
  - `computeDailyLimit(weekId)`: sum(allocated-spent for non-bill buckets) / max(days_remaining, 1)
  - `computeOpeningDailyLimit(weekId)`: sum(opening_allocated_amount for non-bill) / 7

- `src/lib/engine/reroute.ts` (see T2.4)

Deficit trigger conditions (from spec):
- Condition A: `distributable < income * 0.08`
- Condition B: `distributable - (income * 0.05) - (income * 0.03) < food_weekly_minimum`
