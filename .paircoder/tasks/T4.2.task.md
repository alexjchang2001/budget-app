# Objective

Implement bottom tab navigation shell (Home, Buckets, Projection), auth guard, setup guard, and uncategorized count badge on the Buckets tab.

# Implementation Plan

- `src/app/(app)/layout.tsx`:
  - Wraps all three app screens with `<TabBar />`
  - Passes uncategorized count to TabBar via server component fetch from `/api/transactions/uncategorized-count`

- `src/components/nav/TabBar.tsx`:
  - Three tabs: Home (`/`), Buckets (`/buckets`), Projection (`/projection`)
  - Active tab highlighted (use `usePathname()`)
  - Buckets tab shows numeric badge when `uncategorizedCount > 0`
  - Tab bar hidden on `/setup` and `/login` routes (not inside `(app)` group)

- `src/middleware.ts`:
  - Reads JWT from `HttpOnly` cookie
  - Missing/expired JWT → redirect to `/login`
  - Valid JWT but `setup_complete = false` → redirect to `/setup`
  - Applies to all routes under `/(app)`

- `src/app/api/transactions/uncategorized-count/route.ts`:
  - GET: returns `{ count: number }` — transactions where `bucket_id = null` and current week

# Acceptance Criteria

- [ ] `(app)/layout.tsx` wraps all three screens with tab bar; hidden on `/setup` and `/login`
- [ ] Buckets tab shows badge when uncategorized count > 0
- [ ] `src/middleware.ts`: missing/expired JWT → `/login`; `setup_complete = false` → `/setup`
- [ ] Active tab highlighted correctly
- [ ] `bpsai-pair arch check src/app/(app)/` passes

# Verification

- Navigate between tabs → active state updates
- Uncategorized transaction exists → badge on Buckets tab
- Delete session cookie → redirect to `/login`
- Fresh user with `setup_complete = false` → redirect to `/setup`
