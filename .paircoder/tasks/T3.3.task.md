# Objective

Build the Buckets screen with bills bucket rendered as an individual-row list, non-bill bucket cards with color-coded progress bars, expandable transaction lists with confidence badges, classification bottom sheet for reassignment, and uncategorized section.

# Implementation Plan

**`BillsBucket.tsx`:** Renders bills bucket distinctly — NOT as a progress bar:
- Each bill: name, amount, due date, status indicator
- Grey dot = unpaid, green checkmark = confirmed
- "Mark as paid" always visible and tappable
- "Teller confirmed [date]" replaces button on `confirmed_by = 'teller'`
- "Marked paid [date]" on `confirmed_by = 'user'`
- Tap calls `POST /api/bills/{billId}/confirm`

**`BucketCard.tsx`:** Non-bill buckets:
- Label, progress bar (filled = spent/allocated), "$X spent · $Y remaining", % used
- Color thresholds (Screen 2 axis — uses spent/allocated ratio):
  - Green: `< 60%` of weekly allocation spent
  - Amber: `60–89%`
  - Red: `≥ 90%`
- NOTE: These thresholds differ from Screen 1 — do NOT share color utility

**`TransactionRow.tsx`:** Shown on bucket card expand:
- Merchant name, amount, date, bucket label
- Confidence indicator if `classification_confidence < 0.85` (e.g., "85% sure — Food")
- "Confirm?" badge for 0.60–0.84 range

**`ClassifyBottomSheet.tsx`:** Opened on flagged transaction tap:
- Shows current assignment, all buckets listed
- Selection calls override endpoint; sets `classification_override = true`

**Uncategorized section:** Transactions with `confidence < 0.60`; user must assign manually

**Footer:** "Rounding buffer: $X.XX" in muted small text — display only

# Acceptance Criteria

- [ ] Bills bucket: individual rows (not progress bar); "Mark as paid" always visible
- [ ] Status displays: grey dot (unpaid), "Teller confirmed [date]", "Marked paid [date]"
- [ ] Non-bill bucket color: green < 60% spent, amber 60–89%, red ≥ 90% (Screen 2 axis, independent)
- [ ] Transaction expansion shows confidence badge if < 0.85; "Confirm?" badge for 0.60–0.84
- [ ] Classification bottom sheet: bucket list for reassignment; override stored
- [ ] Uncategorized section for `confidence < 0.60`; count badge on tab
- [ ] Rounding buffer shown as muted footer text

# Verification

- Bill shown as grey dot → tap "Mark as paid" → green checkmark appears
- Bucket at 65% spent → amber color
- Flagged transaction → "Confirm?" badge → tap → bottom sheet opens → select bucket → badge disappears
- Uncategorized transaction count badge updates on Buckets tab
