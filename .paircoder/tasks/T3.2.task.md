# Objective

Build the Home screen with the large color-coded daily limit number, three secondary pills, payday countdown, and all conditional states: deficit modal, deposit states, false-positive correction, manual deposit confirmation, Teller error banners, and offline cached state.

# Implementation Plan

**`DailyLimit.tsx`:**
- Large centered number computed from `daily_limit` API
- Color thresholds (Screen 1 axis — uses `opening_allocated_amount` snapshot):
  - Green: `daily_limit ≥ 30%` of opening daily limit
  - Amber: `10–29%`
  - Red: `< 10%`
- Opening daily limit = `sum(opening_allocated_amount for non-bill) / 7`
- NOTE: These thresholds are different from Screen 2 — do NOT share color utility functions

**`PaydayCountdown.tsx`:**
- "Payday in N days" / "Payday — waiting on deposit" / "This week: $X deposited"
- Income line: actual amount once cleared; "Projected ~$low–$high" before

**`DeficitModal.tsx`:**
- Appears when `week.deficit_plan = null` and deficit triggered
- Cannot be dismissed without selection (no X button, backdrop tap non-dismissable)
- Shows full dollar breakdown for each plan
- On selection: `POST /api/weeks/{weekId}/deficit-plan`

**`DepositBanners.tsx`:**
- False-positive: "Wrong deposit detected" → `POST /api/transactions/{id}/correct-deposit`
- Manual confirmation: no deposit by end-of-Friday → banner → bottom sheet showing credits from past 48 hours → selection runs engine
- Empty state: "Waiting for first deposit — showing estimate"
- Teller error: "Bank sync issue — tap to reconnect" (non-blocking)

**Offline:**
- `useOfflineStatus` hook (from T4.1); banner: "Last updated X min ago — no connection"
- Daily limit still visible with cached values + timestamp

# Acceptance Criteria

- [ ] Daily limit color: green ≥ 30%, amber 10–29%, red < 10% of opening daily limit; uses `opening_allocated_amount`
- [ ] Three pills: Bills X/Y paid (both teller + manually confirmed), Debt % contributed, Savings % contributed
- [ ] Deficit modal: undismissable without plan selection; shows dollar breakdown
- [ ] Deficit banner persistent throughout week; taps navigate to Screen 2
- [ ] False-positive banner + manual deposit confirmation flow both working
- [ ] Empty state banner shown for provisional week
- [ ] Offline: non-blocking banner; daily limit still visible

# Verification

- Provisional week → "Waiting for first deposit" banner visible
- Deficit week → modal appears; cannot close without selecting a plan
- Bill confirmed → pills update
- Disconnect network → offline banner appears; number still shows
