# Objective

Manually trace and verify the critical path from fresh install through the first payday week, covering all major engine scenarios. This is not a test suite — it's a structured smoke check before MVP is considered complete.

# Implementation Plan

Trace each scenario in order, document results in `docs/flow-validation.md`:

1. **Setup → provisional week**: Fresh install → complete 5 setup steps → verify provisional `week` row with correct Friday/Thursday boundaries and `status = 'projected'`

2. **First deposit detection**: POST a qualifying Teller webhook → verify Friday reassignment → allocation engine runs → `week.status = 'active'` → Screen 1 shows real daily limit

3. **Deficit Condition A**: Set income $100, bills $93 → engine run → deficit fires (bills > 92% income)

4. **Deficit Condition B**: Set income $1000, bills $900 → engine run → deficit fires (< $50 for food after floors)

5. **Deficit modal**: Verify modal appears and cannot be dismissed without plan selection

6. **Bill confirmation**: Manually confirm a bill → re-routing runs → Debt bucket increases → daily limit recalculates

7. **Classification cold-start**: Insert a transaction → verify all 10 few-shot examples come from `seed_examples` (no user history yet)

8. **All three deficit plans**: Compute Optimal/Emergency/Long-term for the same week → verify each sums exactly to `distributable`

9. **Insolvent state**: Income $100, bills $200 → verify insolvent state (no plan selection modal)

10. **Arch check**: `bpsai-pair arch check src/` → 0 errors, 0 warnings
