# Objective

Implement WebAuthn registration (passkey enrollment) and authentication (assertion) as Next.js API routes, issuing a 7-day JWT on successful assertion. Use `@simplewebauthn/server` and `@simplewebauthn/browser`.

# Implementation Plan

- Install: `@simplewebauthn/server`, `@simplewebauthn/browser`, `jose` (JWT)
- `src/lib/jwt.ts`: sign/verify HS256 JWT with 7-day expiry; `user_id` claim
- `src/lib/auth.ts`: middleware helper that verifies JWT from HttpOnly cookie; used by all downstream routes
- Registration routes:
  - `POST /api/auth/register/challenge`: generate + store challenge in session/DB; return `PublicKeyCredentialCreationOptions`
  - `POST /api/auth/register/verify`: verify credential; create `user` row; store `passkey_credential_id`; set `setup_complete = false`
- Login routes:
  - `POST /api/auth/login/challenge`: fetch user by email/username; generate assertion challenge
  - `POST /api/auth/login/verify`: verify assertion; issue JWT in HttpOnly Secure cookie; redirect to `/` or `/setup`
- Login page `src/app/(auth)/login/page.tsx` with passkey prompt

# Acceptance Criteria

- [x] `POST /api/auth/register/challenge` and `/verify` enroll passkey; `passkey_credential_id` stored on `user`
- [x] `POST /api/auth/login/challenge` and `/verify` assert credential; issue 7-day HS256 JWT as HttpOnly cookie
- [x] `src/lib/auth.ts` middleware helper verifies JWT; returns `{ userId }` or throws on invalid/expired
- [x] Silent re-auth on expiry: bottom sheet appears preserving screen state (no full navigation)
- [x] `bpsai-pair arch check src/app/api/auth/` passes

# Verification

- Login → JWT cookie set → protected route accessible
- Expired JWT → bottom sheet appears, re-auth restores session
