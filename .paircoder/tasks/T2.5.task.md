# Objective

Implement computation for all three deficit plans (Optimal, Emergency, Long-term Responsible with Emergency fallback) and the Insolvent state check. Each returns a full dollar-by-dollar breakdown for modal display.

# Implementation Plan

These functions live in `src/lib/engine/deficit.ts` alongside `checkDeficitTrigger` from T2.2.

**`computeOptimal(income, billTotal, buckets)`:**
- Bills: 100% of fixed total
- Debt: full user-set allocation if distributable allows, otherwise floor (5%)
- Savings: full user-set allocation if distributable allows, otherwise floor (3%)
- Food: trimmed proportionally to close gap
- Flex: cut first, may reach $0

**`computeEmergency(income, billTotal)`:**
- Bills: 100%
- Debt: `income * 0.05`
- Savings: `income * 0.03`
- Food: `income - billTotal - debtFloor - savingsFloor` (minimum $50)
- Flex: $0

**`computeLongTermResponsible(income, billTotal, buckets, foodMin)`:**
1. uniform_cut_pct = target_reduction / sum(normal non-bill allocations)
2. Apply cut; enforce floors; carry overage
3. Distribute overage: Flex first, then Food
4. If Flex+Food at floors and deficit unresolved: return `{ fallback: 'emergency', reason: 'floors_insufficient' }`
5. Verify: sum(all proposed) === distributable; adjust Flex by residue

**`checkInsolvent(income, billTotal, debtFloor, savingsFloor, foodMin)`:**
- Returns true if: `income < billTotal + (income * 0.05) + (income * 0.03) + foodMin`

All functions return: `{ billsAmount, debtAmount, savingsAmount, foodAmount, flexAmount }` (cents).

`src/app/api/weeks/[weekId]/deficit-plan/route.ts`:
- POST: stores `week.deficit_plan`; `deficit_plan_expires_at = week_end`
- Second deposit mid-week: override `deficit_plan_expires_at = now()` and re-run allocation
