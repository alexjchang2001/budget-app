# Objective

Implement the `POST /api/schedule/parse` route: upload image to Supabase Storage, call Claude vision API with spec prompt, return parsed `{ shift_count, shift_days, confidence }`. Implement confirmation endpoint to create `schedule_parse` record and update `week` projection.

# Implementation Plan

- `src/lib/vision/schedule-parser.ts`:
  - `parseScheduleImage(imageBuffer, mimeType)`: calls Claude API with vision; uses exact prompt from spec:
    ```
    You are parsing a restaurant employee scheduling app screenshot.
    Extract the total number of shifts and the day of week for each shift.
    ...
    Return JSON only: { "shift_count": <int>, "shift_days": <array>, "confidence": <float 0-1> }
    ```
  - Strict JSON parse; return `{ shift_count, shift_days, confidence }` or throw `ParseFailedError`

- `src/app/api/schedule/parse/route.ts`:
  - Accept multipart form data with image file
  - Upload to Supabase Storage bucket `schedule-screenshots`
  - Call `parseScheduleImage`; create pending `schedule_parse` row with `confirmed_by_user = false`
  - Return `{ shift_count, shift_days, confidence, parse_id }` or `{ error: 'parse_failed' }`

- `src/app/api/schedule/confirm/route.ts`:
  - POST: `{ parse_id, per_shift_min, per_shift_max, week_id }`
  - Validate `per_shift_min < per_shift_max`, both > 0
  - Compute `projected_low = per_shift_min * shift_count`, `projected_high = per_shift_max * shift_count`
  - Update `schedule_parse.confirmed_by_user = true`
  - Update `week.income_projected_low` and `week.income_projected_high`

# Acceptance Criteria

- [ ] Image stored in Supabase Storage `schedule-screenshots`; URL in `schedule_parse.raw_screenshot_url`
- [ ] Claude vision uses exact prompt from spec; response parsed strictly as `{ shift_count, shift_days, confidence }`
- [ ] Returns `{ shift_count, shift_days, confidence, parse_id }` on success; `{ error: 'parse_failed' }` on exception
- [ ] `/api/schedule/confirm` validates `per_shift_min < per_shift_max`; updates `schedule_parse` + `week`
- [ ] `bpsai-pair arch check src/app/api/schedule/` passes

# Verification

- Upload valid schedule image → parsed result returned; `schedule_parse` row created in DB
- Confirm with valid range → `week.income_projected_low/high` updated
- Upload corrupted image → `{ error: 'parse_failed' }` returned; no DB writes
