# Objective

Implement server-side re-routing on `bill_status` change to confirmed: subtract freed bill amount from bills bucket, redistribute to non-bill buckets in priority order (Debt→Savings→Food→Flex), recalculate daily limit.

# Implementation Plan

- `src/lib/engine/reroute.ts`:
  - `rerouteFreedBillAmount(weekId, billId, freedAmount)`: retrieves non-bill buckets in priority_order; routes full freedAmount to first bucket (no partial split in MVP); edge case: no non-bill buckets → Savings
  - `getPriorityOrderedBuckets(weekId)`: returns non-bill buckets sorted Debt(1)→Savings(2)→Food(3)→Flex(4)→custom
  - Priority order hard-coded in MVP — not user-configurable

- `src/lib/engine/daily-limit.ts`:
  - `computeDailyLimit(weekId)`: `sum(allocated - spent for non-bill) / max(daysRemainingInWeek, 1)`
  - Where daysRemainingInWeek = days from today through Thursday inclusive

- `src/app/api/bills/[billId]/confirm/route.ts`:
  - POST: accepts `{ confirmed_by: 'user' | 'teller', transaction_id?: string }`
  - Updates `bill_status`; calls `rerouteFreedBillAmount`; returns updated week state
  - All mutations in a single DB transaction

# Acceptance Criteria

- [x] Priority order hard-coded: Debt(1)→Savings(2)→Food(3)→Flex(4)→custom by `priority_order` ascending
- [x] Full `freed_amount` routed to first eligible bucket; if none, falls to Savings
- [x] Daily limit recomputed after re-routing: `sum(allocated - spent, non-bill) / max(days_remaining, 1)`
- [x] `POST /api/bills/{billId}/confirm` triggers re-routing and returns updated state
- [x] All mutations in a single DB transaction

# Verification

- Bill confirmed → bills `bucket_allocation.allocated_amount` decremented by `bill.amount`
- Freed amount appears in Debt bucket (highest priority)
- Daily limit updates immediately after confirmation
- All DB state consistent (no partial writes)
