# Objective

Register a service worker for offline app shell caching and API fallback. This is a named build task from the spec â€” push notification logic must NOT be wired (deferred to Iteration 2), but the registration architecture must support it additively.

# Implementation Plan

- `public/sw.js`: service worker with two cache strategies:
  - Cache-first for app shell assets (JS bundles, CSS, icons)
  - Network-first with cache fallback for `/api/weeks/current` and `/api/buckets`
  - Explicit comment at push stub point: `// PUSH_STUB: push event listener added in Iteration 2`
  - Service worker must NOT register push or requestNotificationPermission

- Register in `src/app/layout.tsx` root layout:
  ```js
  if ('serviceWorker' in navigator) {
    navigator.serviceWorker.register('/sw.js')
  }
  ```

- `src/hooks/useOfflineStatus.ts`:
  - `useOfflineStatus()`: returns `{ isOffline, lastUpdated }`
  - Listens to `online`/`offline` events; tracks last successful API response timestamp

- `src/components/ui/OfflineBanner.tsx`:
  - Non-blocking top strip: "Last updated X min ago â€” no connection"
  - Only shown when `isOffline = true`
  - Positioned above app content, not over it

- `public/manifest.json` (update from T1.1):
  - `display: standalone`, `start_url: /`, `theme_color: #000000`, `background_color: #000000`
  - At least one 192Ă—192 icon entry

- PWA install: `beforeinstallprompt` event captured in root layout; prompt shown on setup step 5 completion
