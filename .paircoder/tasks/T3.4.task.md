# Objective

Build the Projection screen: current income projection display, schedule upload → Claude vision parse → confirmation card flow (with manual fallback), and closed weeks history list with tap-to-expand bucket breakdown.

# Implementation Plan

**Top section — projection:**
- Current projection line: "Next week: $low–$high · Budget baseline: $baseline"
- "Update from schedule" button → native image picker (HTML `<input type="file" accept="image/*">`)

**`ScheduleUpload.tsx`:**
- Handles image selection and `POST /api/schedule/parse`
- Shows loading state during parse

**`ParseConfirmCard.tsx`:**
- Parse success (confidence ≥ 0.70): shifts listed (e.g. "4 shifts — Mon, Wed, Fri, Sat"), per-shift range (pre-filled from most recent `schedule_parse`, editable), projected totals (Low/High/Baseline)
- Confirm → `POST /api/schedule/confirm`; Edit → allow range modification before confirming
- Parse low-confidence (< 0.70): "Couldn't read your schedule clearly — enter shifts manually" + integer shift count input + range; same confirmation flow
- Parse failure: "Something went wrong — try a clearer screenshot or enter shifts manually"
- Manual fallback always available regardless of confidence

**`WeekHistoryRow.tsx`:**
- Each closed week: date range, `income_actual`, `deficit_plan` (if any), total spent vs allocated
- Tap to expand: per-bucket breakdown for that week
- Empty state: "Your history will appear here after your first full week"

# Acceptance Criteria

- [ ] "Update from schedule" → image picker → `POST /api/schedule/parse`
- [ ] Confidence ≥ 0.70: confirmation card with shifts, editable range, projected low/high/baseline; Confirm updates `week.income_projected_low/high`
- [ ] Confidence < 0.70: "Couldn't read" message + manual shift count + range input; same confirmation flow
- [ ] Parse failure: error + manual fallback
- [ ] History: closed weeks newest first; date range, income, deficit plan, totals
- [ ] Tap expand: bucket breakdown for that week
- [ ] Empty state: "Your history will appear here after your first full week"

# Verification

- Upload clear schedule screenshot → parse result shown; Confirm → projection updates
- Upload blurry screenshot (mock < 0.70 confidence) → manual input shown
- Two closed weeks exist → history list shows both newest first; expand shows buckets
