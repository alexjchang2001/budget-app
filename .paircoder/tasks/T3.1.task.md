# Objective

Build the 5-step setup wizard as a single page with step state management. On completion: provisional week created, PWA install banner shown, navigate to Screen 1.

# Implementation Plan

Each step in its own component file (each ≤ 200 lines):

**Step1Passkey.tsx:**
- `startRegistration` from `@simplewebauthn/browser`
- Recovery email input (required)
- 16-char recovery code displayed with warning copy; "I've saved my code" checkbox blocks Next

**Step2TellerConnect.tsx:**
- Render Teller Connect JS SDK in bottom sheet
- On success: POST enrollment ID + access token to `/api/setup/teller-connect`
- Retry button on SDK error

**Step3Bills.tsx:**
- Dynamic list of bill rows: name, amount ($), due day (1–31)
- Inline validation: amount > 0, due day 1–31, name not blank
- At least one bill required to proceed

**Step4Buckets.tsx:**
- Pre-populated rows: Debt, Savings, Food, Flex (bills not shown here — handled by fixed amounts)
- Each row: name, % input
- Running total shown; "X% unallocated" error blocks Next
- Flex is the only removable default; user can rename or add custom buckets

**Step5Income.tsx:**
- Baseline weekly income ($)
- Per-shift range: min and max ($); validation: min < max, both > 0
- These seed `user.baseline_weekly_income` and initial `schedule_parse` range

**Completion:**
- `POST /api/setup/complete`: saves all data, creates provisional week (allocation engine runs), sets `user.setup_complete = true`
- PWA install banner shown (`beforeinstallprompt`)
- Navigate to `/`

# Acceptance Criteria

- [ ] Step 1: passkey enrolled; recovery code shown once; checkbox required to proceed
- [ ] Step 2: Teller Connect bottom sheet; enrollment stored; retry on error
- [ ] Step 3: bill rows with inline validation; ≥ 1 bill required
- [ ] Step 4: running total shown; "X% unallocated" error; Flex only removable default
- [ ] Step 5: income baseline + per-shift range; min < max validation
- [ ] On completion: provisional week created; `setup_complete = true`; navigate to `/`

# Verification

- Complete all 5 steps → land on Screen 1 with "Waiting for first deposit" banner
- Attempt to proceed without required fields → error shown, progression blocked
- Recovery code shown exactly once on Step 1
